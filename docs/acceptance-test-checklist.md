# Acceptance Test Checklist

## Student Flow
- [ ] student can log in
- [ ] student can see problem list
- [ ] student can open a problem detail page
- [ ] student can write code in editor
- [ ] student can run code
- [ ] run result is shown on screen
- [ ] run result is saved as submission
- [ ] student can request AI hint
- [ ] AI response is shown on screen
- [ ] AI interaction is logged
- [ ] student submission history can be viewed later

## Teacher Flow
- [ ] teacher can log in
- [ ] teacher can see problem list
- [ ] teacher can see problem details
- [ ] teacher can work with seeded/sample problems in early MVP

## Admin / Exam Mode
- [ ] exam mode flag can be toggled
- [ ] system can read exam mode flag correctly
- [ ] exam mode restrictions can be extended later

## General
- [ ] no console-breaking backend/database errors during main flows
- [ ] database-backed flows work with seeded data
- [ ] core MVP flows work end-to-end with real database records
